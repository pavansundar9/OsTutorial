chmod command:


Linux chmod command is used to change the access permissions of files and directories. It stands for change mode. It can not change the permission of symbolic links. Even, it ignores the symbolic links come across recursive directory traversal.


The basic syntax of chmod command is as follows:


chmod <options> <permissions> <file name>


In the Linux file system, each file is associated with a particular owner and have permission access for different users. The user classes may be:

  • owner(u): file's owner
  • group member(g): users who are members of the file's group
  • Others (Everybody else): users who are neither the file's owner nor members of the file's group


    • The file permissions in Linux are the following three types:

  • read(r): Permission to read the file.
  • write(w): Permission to write (or delete) the file.
  • execute(x): Permission to execute the file, or, in the case of a directory, search it.


    • -c, --changes
      : It is similar to the verbose option, but the difference is that it is reported if a change has been made.

    • -f, --silent, --quiet
      :It is used to suppress the error messages.

    • -v, --verbose
      : It is used to display a diagnostic for every processed file.

    • --no-preserve-root
      :It is used for not treating the backslash symbol ('/'), especially (the default).
